# 意图引导

> 当用户输入 `/reqplan guide`、`/reqplan start` 或意图不明确时，按以下流程引导用户。

## 首次启动引导

用户第一次触发 ReqPlan-v3 或输入 `/reqplan start` 时，先展示能力概览：

```
🔄 ReqPlan-v3 已激活 — 项目全生命周期管理引擎

我支持以下 3 种模式，请选择或直接描述您的需求：

1️⃣ 开发模式
   新功能开发、模块迭代、架构重构
   流程：需求分析 → 设计 → 实施 → 验证

2️⃣ 分析模式
   代码审查、设计评审、质量评估
   流程：明确目标 → 划定范围 → 逐项分析 → 输出报告

3️⃣ 修复模式
   Bug 排查、问题诊断、异常修复
   流程：收集信息 → 诊断根因 → 修复 → 验证

您可以说：
  "帮我开发一个用户登录功能"
  "帮我审查一下这段代码"
  "系统报错了，帮我看看"
  /reqplan guide    → 查看详细引导
  /reqplan init     → 初始化 Harness 目录
  /reqplan status   → 查看当前状态
```

> 用户选择场景后，自动进入对应流程的 Step 1 引导。不需要一次问完所有问题。

## 命令行为：/reqplan guide

**触发时机**：用户输入 `/reqplan guide`，或 AI 判断用户意图模糊需要引导时。

**执行步骤**：
1. 确认当前 Skill 是否应激活（见下方激活确认检查表）
2. 如果应激活但意图模糊 → 按"三大场景"分类引导
3. 如果用户完全说不清 → 展示三选一兜底引导

**关键原则**：不需要一次问完所有问题。先理解核心意图，用户说多少问多少，逐步深入。

---

## 激活确认（每次都应执行）

**在执行任何引导之前，先确认本 skill 是否应处于激活状态：**

```
当前用户的输入是否属于以下任一场景？
  □ 开发类：用户想开发/实现/新增/创建/设计/构建一个功能、模块、页面、API、系统
  □ 分析类：用户想审查/检查/分析/评估/审计代码、设计、架构、质量
  □ 修复类：用户报告Bug/错误/异常/崩溃/故障，或说"不工作了"、"出错了"
  □ 不确定性：用户说"帮我规划一下"、"怎么做"、"有什么方案"、"下一步做什么"
  □ 🪞 元任务类：用户要求审查/检查/分析/修复本 Skill（reqplan-v3）自身

命中任一 → 本 skill 继续生效，按对应场景引导
全部未命中 → 本 skill 不应激活，用通用能力回应用户
```

> **🪞 元任务处理说明**：
> - 元任务（审查/修复 Skill 自身）同样受本 Skill 状态机约束
> - 不得以"审查Skill本身"为由跳过入口清单或状态机
> - 元任务执行流程详见 chunk-02-flows.md → 流程4（自绑定审查）

> 如果确认本 skill 应激活但用户意图仍模糊，按以下流程逐步引导：

## 用户意图分类

快速匹配三条判别线：

```
"我要做/开发/实现/新增/改进/重构一个..." → 开发场景
"帮我看看/审查/分析/评估/检查..."       → 分析场景
"出错了/报错了/不工作了/Bug/崩溃/异常..." → 修复场景
```

如果用户输入模糊，按以下流程引导：

---

## 场景1：开发引导

按顺序引导用户澄清：

### 1. 核心功能

理解这个功能解决什么问题。先问目标，再问细节：

```
我理解您需要开发新功能。请先告诉我：
这个功能主要解决什么问题？用户通过它完成什么操作？
```

### 2. 涉及角色

```
谁会使用这个功能？大致分为几类角色？
（如管理员、普通用户、访客等）
```

### 3. 数据与边界

```
这个功能涉及哪些核心数据？
技术上有什么约束？（框架、性能、安全性、兼容性等）
```

> 不需要一次问完所有问题。先理解核心意图，用户说多少问多少，逐步深入。

---

## 场景2：分析引导

### 1. 明确目标

```
您要分析什么？代码质量、架构设计、安全性、还是性能？
```

### 2. 划定范围

```
分析范围是什么？是整个项目、某个模块、还是特定文件？
```

### 3. 关注维度

质量审查关注：命名规范、代码结构、复杂度、重复代码
安全审查关注：输入验证、权限控制、敏感信息泄露
性能审查关注：慢查询、大循环、不必要的渲染

---

## 场景3：修复引导

按信息收集 > 诊断 > 方案的顺序推进：

### 1. 收集现象

```
请描述问题现象：什么时候出现？看到了什么报错？
是每次必现还是偶发？
```

### 2. 环境信息

```
运行环境是什么？（系统、浏览器、应用版本）
最近做了什么变更？（部署、改代码、更新依赖）
```

### 3. 影响评估

```
影响范围多大？影响多少用户？
有没有临时变通方案？
```

---

```
   从问题收集到诊断根因、修复方案、验证修复

请选择，或直接描述您需要什么帮助：
```
