# ReqPlan 故障排查指南

## 常见问题

### 问题1: Skill无法触发

**现象**：输入命令或自然语言后，Skill没有响应

**可能原因**：
1. 触发词不匹配
2. 命令格式不正确
3. Skill未正确安装
4. 上下文过期

**解决方法**：
```
1. 检查命令格式是否正确：
   - 正确格式：/reqplan <command>
   - 示例：/reqplan start

2. 尝试使用其他触发方式：
   - 自然语言："我需要一个订单管理模块"
   - 命令方式：/reqplan intent 订单管理模块

3. 检查Skill是否已安装：
   - 查看已安装的Skill列表

4. 如果会话需要重新开始：
   - 使用 /reqplan start 重新开始
```

### 问题2: 状态文件损坏

**现象**：无法读取或写入状态

**错误码**：E701

**解决方法**：
```
1. 系统会自动尝试从备份恢复
2. 如果备份也损坏：
   - 使用 /reqplan start 重新初始化项目
   - 手动恢复快照文件

3. 预防措施：
   - 定期创建快照
   - 启用自动备份
```

### 问题3: 项目未初始化

**现象**：执行命令时提示"项目不存在"

**错误码**：E702

**解决方法**：
```
1. 使用 /reqplan start 创建新项目
2. 提供项目名称和基本信息
3. 按照引导完成初始化
```

### 问题4: 文件写入失败

**现象**：无法保存文档或状态

**错误码**：E602

**可能原因**：
1. 磁盘空间不足
2. 文件权限不足
3. 目录不存在

**解决方法**：
```
1. 检查磁盘空间
2. 检查文件/目录权限
3. 确保路径正确
4. 尝试以管理员身份运行
```

### 问题5: 会话需要重新开始

**现象**：提示"会话已过期"或需要重新开始

**错误码**：E703

**解决方法**：
```
1. 系统会列出最近的需求供选择
2. 选择需要继续的需求
3. 或者创建新需求
```

### 问题6: 任务不存在

**现象**：无法找到指定的任务

**错误码**：E202

**解决方法**：
```
1. 使用 /reqplan task list 查看所有任务
2. 确认任务ID是否正确
3. 检查任务是否已被删除
```

### 问题7: 模板不存在

**现象**：文档生成失败

**错误码**：E101

**解决方法**：
```
1. 系统会自动使用默认格式
2. 检查模板文件是否存在
3. 确保模板路径正确
```

### 问题8: 验收失败

**现象**：测试用例未通过

**解决方法**：
```
1. 查看失败原因
2. 修复代码或配置
3. 重新执行验收测试
4. 使用 /reqplan verify 重新测试
```

### 问题9: 流程切换失败

**现象**：无法切换到指定流程

**错误码**：E801

**解决方法**：
```
1. 检查流程名称是否正确
2. 使用 /reqplan flow list 查看可用流程
3. 确认当前状态是否支持该流程切换
4. 查看流程历史了解当前位置
```

### 问题10: 文件同步检测失败

**现象**：无法检测文件变更或影响分析

**错误码**：E802

**解决方法**：
```
1. 使用 /reqplan sync 手动触发同步检测
2. 检查文件路径是否正确
3. 确认文件是否可读
4. 查看同步报告了解详细信息
```

## v3特定问题

### 问题11: 智能引导不工作

**现象**：引导信息不显示或不正确

**解决方法**：
```
1. 使用 /reqplan guide 手动触发引导
2. 确认当前状态有效
3. 检查流程配置是否正确
4. 查看状态概览了解当前进度
```

## 调试技巧

### 查看状态
```
/reqplan status          # 查看全局状态
/reqplan task list       # 查看任务列表
/reqplan flow current    # 查看当前流程
```

### 日志查看
```
检查以下路径的日志文件：
- .trae/reqplan/logs/
- .trae/reqplan/projects/<project>/logs/
```

### 手动恢复

**恢复状态文件**：
```yaml
1. 找到备份文件：state.backup.yaml
2. 复制到 state.yaml
3. 重新启动Skill
```

**恢复快照**：
```yaml
1. 找到快照文件：snapshots/snapshot-<timestamp>.yaml
2. 使用 /reqplan restore <snapshotPath>
3. 或手动复制到 state.yaml
```

**重置流程**：
```
1. 使用 /reqplan flow list 查看可用流程
2. 选择合适的流程重新开始
3. 或使用 /reqplan start 创建全新项目
```
