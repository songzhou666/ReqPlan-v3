# 文档完善流程 (Documentation Improvement Flow)

## 流程概述
生成、完善、更新项目的技术文档和用户文档。
**Harness 适配**：L / M / H — L 级别走简化 3 阶段管道（入口 → 执行 → 回写），M/H 走完整 5 阶段。

## 适用场景
- 技术文档生成
- 用户文档编写
- API文档生成
- 文档质量提升
- 文档标准化

## 流程步骤

### 步骤1：文档需求分析
**目标**：分析需要哪些文档
**引导**：
```
📋 文档完善 - 需求分析

分析文档需求：
- 项目当前文档状态
- 需要生成的文档类型
- 文档受众分析
- 文档优先级

当前文档状态：
✅ 需求文档 (req.md)
✅ 设计文档 (design.md)
❌ API文档 (缺失)
❌ 用户手册 (缺失)
❌ 部署文档 (缺失)

输入 /reqplan docs 开始分析
```
**输出**：文档清单
**writeback_target**：`docs/（各文档目录）`
**完成检查**：文档需求明确

### 步骤2：技术文档生成
**目标**：生成技术相关文档
**引导**：
```
🏗️ 技术文档生成

生成技术文档：
- 架构设计文档
- 技术选型说明
- 数据库设计文档
- 接口设计文档
- 部署架构文档

输入 /reqplan docs 生成技术文档
```
**输出**：技术文档集
**writeback_target**：`docs/（各文档目录）`
**完成检查**：技术文档生成完成

### 步骤3：用户文档生成
**目标**：生成用户使用文档
**引导**：
```
👥 用户文档生成

生成用户文档：
- 用户手册
- 操作指南
- 常见问题FAQ
- 最佳实践
- 快速入门

输入 /reqplan docs 生成用户文档
```
**输出**：用户文档集
**writeback_target**：`docs/（各文档目录）`
**完成检查**：用户文档生成完成

### 步骤4：API文档生成
**目标**：生成API接口文档
**引导**：
```
🔌 API文档生成

生成API文档：
- 接口列表
- 请求/响应示例
- 参数说明
- 错误码说明
- 使用示例

输入 /reqplan docs 生成API文档
```
**输出**：API文档
**writeback_target**：`docs/（各文档目录）`
**完成检查**：API文档生成完成

### 步骤5：文档审核
**目标**：审核文档质量
**引导**：
```
🔍 文档审核

审核文档质量：
- 内容完整性
- 准确性检查
- 格式规范
- 语言表达
- 示例正确性

输入 /reqplan review 审核文档
```
**输出**：审核报告
**writeback_target**：`docs/harness/review-{{taskId}}.md`
**完成检查**：文档审核完成

### 步骤6：文档优化
**目标**：根据审核优化文档
**引导**：
```
✏️ 文档优化

根据审核意见优化：
- 补充缺失内容
- 修正错误
- 改进表述
- 完善示例
- 统一格式

输入 /reqplan docs 优化文档
```
**输出**：优化后的文档
**writeback_target**：`docs/（各文档目录）`
**完成检查**：文档优化完成

### 步骤7：文档发布
**目标**：发布最终文档
**引导**：
```
🚀 文档发布

文档发布准备：
- 最终版本确认
- 文档版本标记
- 发布位置选择
- 通知相关人员

输入 /reqplan docs 完成发布
```
**输出**：文档发布完成标记
**writeback_target**：`AGENTS.md（更新文档入口）`
**完成检查**：文档发布完成

## 流程进度
```
步骤进度：1/7 完成
总体进度：14%
```

## 分支选项
- 可选择生成部分类型的文档
- 可随时跳转到其他流程
- 支持分阶段发布

## 完成标准
- 文档清单完整
- 技术文档齐全
- 用户文档齐全
- API文档完整
- 文档审核通过
- 文档已发布

## 失败策略
- 文档生成失败时提供模板
- 支持手动编辑文档
- 提供分批次生成选项

## 适用边界

### 适合场景
- 项目文档建立、API文档更新、使用指南编写、架构文档维护

### 轻量处理（可直接自动化）
- 小幅错别字修正

### 需人工介入
- 文档结构决策、发布时机
