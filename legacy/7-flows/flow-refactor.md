# 架构重构流程 (Architecture Refactoring Flow)

## 流程概述
对系统架构进行优化、重构，解决技术债务，提升系统质量和可维护性。
**Harness 适配**：M / H — 重构类任务至少需要标准 Harness 级别；L 级别仅限超轻量代码清洗场景。

## 适用场景
- 系统架构优化
- 技术债务偿还
- 模块重构
- 技术栈升级
- 性能提升重构

## 流程步骤

### 步骤1：架构现状分析
**目标**：分析当前架构状况
**引导**：
```
🏗️ 架构重构 - 现状分析

分析当前架构：
- 架构模式评估
- 技术债务识别
- 性能瓶颈分析
- 可维护性评估
- 扩展性评估

当前状态分析中...
输入 /reqplan design 查看当前架构
```
**输出**：架构现状报告
**writeback_target**：`docs/design/{{taskId}}.md`
**完成检查**：现状分析完成

### 步骤2：重构目标确定
**目标**：确定重构目标和范围
**引导**：
```
🎯 重构目标确定

确定重构目标：
- 性能提升目标
- 可维护性目标
- 技术债务偿还计划
- 重构范围边界
- 成功标准定义

输入 /reqplan design 确定目标
```
**输出**：重构目标文档
**writeback_target**：`docs/design/{{taskId}}.md`
**完成检查**：目标明确

### 步骤3：重构方案设计
**目标**：设计重构方案
**引导**：
```
📐 重构方案设计

设计重构方案：
- 新架构设计
- 迁移策略
- 技术选型
- 模块拆分方案
- 接口调整方案

输入 /reqplan design 设计方案
```
**输出**：重构方案文档
**writeback_target**：`docs/design/{{taskId}}.md`
**完成检查**：方案设计完成

### 步骤4：风险评估
**目标**：评估重构风险
**引导**：
```
⚠️ 风险评估

评估重构风险：
- 技术风险
- 进度风险
- 质量风险
- 业务影响
- 应对措施

输入 /reqplan review 进行风险评估
```
**输出**：风险评估报告
**writeback_target**：`docs/harness/review-{{taskId}}.md`
**完成检查**：风险评估完成

### 步骤5：重构计划
**目标**：制定详细的重构计划
**引导**：
```
📅 重构计划

制定重构计划：
- 阶段划分
- 任务分解
- 时间安排
- 资源分配
- 回滚方案

输入 /reqplan plan 制定计划
```
**输出**：重构计划文档
**writeback_target**：`.agent/plans/plan-{{taskId}}.md`
**完成检查**：计划制定完成

### 步骤6：重构执行
**目标**：按计划执行重构
**引导**：
```
🔨 重构执行

执行重构任务：
- 按阶段执行
- 持续验证
- 小步迭代
- 及时反馈

当前阶段：架构调整
任务进度：1/10 完成

输入 /reqplan task list 查看重构任务
```
**输出**：重构进度、代码变更
**writeback_target**：`PR/MR`
**完成检查**：重构执行完成

### 步骤7：重构验证
**目标**：验证重构效果
**引导**：
```
✅ 重构验证

验证重构效果：
- 功能验证
- 性能测试
- 安全检查
- 兼容性验证
- 用户验收

输入 /reqplan verify 执行验证
```
**输出**：验证报告
**writeback_target**：`docs/test/regression-{{taskId}}.md`
**完成检查**：验证通过

### 步骤8：文档更新
**目标**：更新相关文档
**引导**：
```
📚 文档更新

更新相关文档：
- 架构文档
- 设计文档
- API文档
- 部署文档
- 运维手册

输入 /reqplan docs 更新文档
```
**输出**：更新后的文档
**writeback_target**：`AGENTS.md（更新结构说明）`、`docs/harness/project-constraints.md`、`任务系统（更新状态）`
**完成检查**：文档已更新

## 流程进度
```
步骤进度：1/8 完成
总体进度：12%
```

## 分支选项
- 步骤4后：如风险过高可调整方案
- 步骤6中：可随时暂停和回滚
- 完成后可跳转回其他流程

## 完成标准
- 架构现状分析完整
- 重构目标明确
- 重构方案可行
- 风险评估充分
- 重构计划详细
- 重构执行完成
- 重构验证通过
- 文档已更新

## 失败策略
- 提供回滚方案
- 支持分阶段重构
- 提供暂停和调整选项

## 适用边界

### 适合场景
- 模块重构、架构调整、性能优化、技术债务清理

### 轻量处理（可直接自动化）
- 单文件重命名

### 需人工介入
- 重构范围决策、兼容性策略
