# 设计评审流程 (Design Review Flow)

## 流程概述
对系统架构设计、接口设计、数据模型设计等进行全面评审的流程。
**Harness 适配**：L / M / H — L 级别走简化 3 阶段管道（入口 → 执行 → 回写），M/H 走完整 5 阶段。

## 适用场景
- 新功能设计评审
- 架构方案评审
- API接口评审
- 数据模型评审
- 设计变更评审

## 流程步骤

### 步骤1：设计收集
**目标**：收集待评审的设计文档
**引导**：
```
📋 设计评审 - 收集阶段

请提供或确认待评审的设计：
- 架构设计文档
- API接口设计
- 数据模型设计
- 其他相关设计

自动检测到以下设计文档：
• design/architecture.md
• design/api.md
• design/database.md

输入 /reqplan design 查看设计文档
```
**输出**：设计文档清单
**writeback_target**：`docs/harness/review-design-{{taskId}}.md`
**完成检查**：所有待评审设计已收集

### 步骤2：架构评审
**目标**：评审系统架构设计
**引导**：
```
🏗️ 架构评审阶段

检查以下方面：
- 架构合理性
- 模块划分
- 依赖关系
- 扩展性
- 性能考虑

输入 /reqplan review 执行架构评审
```
**输出**：架构评审报告
**writeback_target**：`docs/harness/review-design-{{taskId}}.md`
**完成检查**：架构评审完成

### 步骤3：接口评审
**目标**：评审API接口设计
**引导**：
```
🔌 接口评审阶段

检查以下方面：
- API设计规范
- 参数设计
- 响应格式
- 错误处理
- 版本管理

输入 /reqplan review 继续接口评审
```
**输出**：接口评审报告
**writeback_target**：`docs/harness/review-design-{{taskId}}.md`
**完成检查**：接口评审完成

### 步骤4：数据模型评审
**目标**：评审数据模型设计
**引导**：
```
🗄️ 数据模型评审阶段

检查以下方面：
- 表结构设计
- 字段定义
- 索引设计
- 关系设计
- 数据一致性

输入 /reqplan review 继续数据模型评审
```
**输出**：数据模型评审报告
**writeback_target**：`docs/harness/review-design-{{taskId}}.md`
**完成检查**：数据模型评审完成

### 步骤5：评审总结
**目标**：汇总所有评审意见
**引导**：
```
📊 评审总结阶段

汇总各方面评审意见：
- 架构评审发现
- 接口评审发现
- 数据模型评审发现
- 优先级排序

输入 /reqplan review 生成总结报告
```
**输出**：评审总结报告
**writeback_target**：`docs/harness/review-design-{{taskId}}.md`
**完成检查**：评审总结完成

### 步骤6：设计修改
**目标**：根据评审意见修改设计
**引导**：
```
✏️ 设计修改阶段

根据评审意见修改设计：
1. 高优先级问题修复
2. 中优先级问题处理
3. 低优先级建议考虑

输入 /reqplan design 编辑设计文档
```
**输出**：更新后的设计文档
**writeback_target**：`docs/design/{{taskId}}.md`
**完成检查**：设计问题已修复

### 步骤7：二次确认
**目标**：确认修改后的设计
**引导**：
```
✅ 二次确认阶段

最终检查：
✅ 所有问题已修复
✅ 设计符合要求
✅ 文档已更新

输入 /reqplan review 完成最终确认
```
**输出**：最终确认标记
**writeback_target**：`任务系统（反馈评审结论）`
**完成检查**：二次确认通过

## 流程进度
```
步骤进度：1/7 完成
总体进度：14%
```

## 分支选项
- 步骤1后：可直接进入任意评审环节
- 步骤5后：可选择是否进行修改
- 步骤7后：可跳转回完整项目或需求迭代流程

## 完成标准
- 所有设计文档已评审
- 评审意见完整记录
- 设计问题已修复
- 二次确认通过

## 失败策略
- 发现严重设计问题时可暂停
- 支持重新设计
- 提供多轮评审选项

## 适用边界

### 适合场景
- 架构设计、接口设计、数据库设计评审

### 轻量处理（可直接自动化）
- 微小设计调整

### 需人工介入
- 架构取舍、技术选型
