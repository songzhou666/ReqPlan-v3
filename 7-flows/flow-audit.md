# 代码审计流程 (Code Audit Flow)

## 流程概述
对现有代码进行全面质量审查，发现潜在问题、安全风险和改进机会。
**Harness 适配**：L / M / H — L 级别走简化 3 阶段管道（入口 → 执行 → 回写），M/H 走完整 5 阶段。

## 适用场景
- 代码质量检查
- 安全审计
- 性能分析
- 技术债务评估
- 代码规范化

## 流程步骤

### 步骤1：代码收集
**目标**：确定审计范围，收集代码
**引导**：
```
📋 代码审计 - 收集阶段

请确定审计范围：
1. 整个项目
2. 特定模块
3. 最近变更的代码
4. 指定文件列表

自动检测到以下代码：
• src/ (整个源码目录)
• modules/user-service/ (特定模块)
• recent-changes/ (最近提交)

选择审计范围，或输入 /reqplan review 开始审计
```
**输出**：审计范围确定
**writeback_target**：`docs/harness/audit-{{taskId}}.md`
**完成检查**：审计范围明确

### 步骤2：静态分析
**目标**：执行静态代码分析
**引导**：
```
🔍 静态分析阶段

执行以下检查：
- 代码规范检查
- 潜在bug检测
- 复杂度分析
- 重复代码检测
- 不良实践识别

正在执行静态分析...
```
**输出**：静态分析报告
**writeback_target**：`docs/harness/audit-{{taskId}}.md`
**完成检查**：静态分析完成

### 步骤3：代码审查
**目标**：进行人工代码审查
**引导**：
```
👀 代码审查阶段

重点审查：
- 代码可读性
- 逻辑正确性
- 异常处理
- 边界条件
- 注释完整性

输入 /reqplan review 进行代码审查
```
**输出**：代码审查报告
**writeback_target**：`docs/harness/audit-{{taskId}}.md`
**完成检查**：代码审查完成

### 步骤4：安全审计
**目标**：进行安全漏洞和风险分析
**引导**：
```
🔒 安全审计阶段

检查安全问题：
- 输入验证
- SQL注入风险
- XSS漏洞
- 认证授权
- 敏感数据处理

输入 /reqplan review 进行安全审计
```
**输出**：安全审计报告
**writeback_target**：`docs/harness/audit-{{taskId}}.md`
**完成检查**：安全审计完成

### 步骤5：性能分析
**目标**：分析性能瓶颈和优化机会
**引导**：
```
⚡ 性能分析阶段

分析性能问题：
- 算法复杂度
- 数据库查询
- 内存使用
- I/O操作
- 缓存策略

输入 /reqplan review 进行性能分析
```
**输出**：性能分析报告
**writeback_target**：`docs/harness/audit-{{taskId}}.md`
**完成检查**：性能分析完成

### 步骤6：审计总结
**目标**：汇总所有审计发现
**引导**：
```
📊 审计总结阶段

汇总所有发现：
- 问题严重程度分类
- 优先级排序
- 影响评估
- 统计数据

输入 /reqplan review 生成审计总结
```
**输出**：审计总结报告
**writeback_target**：`docs/harness/audit-report-{{taskId}}.md`
**完成检查**：审计总结完成

### 步骤7：改进计划
**目标**：制定改进和重构计划
**引导**：
```
📋 改进计划阶段

根据审计发现制定计划：
- 紧急问题修复
- 技术债务偿还
- 代码重构任务
- 优化任务列表

输入 /reqplan plan 制定改进计划
```
**输出**：改进计划文档
**writeback_target**：`docs/harness/audit-report-{{taskId}}.md`
**完成检查**：改进计划制定完成

## 流程进度
```
步骤进度：1/7 完成
总体进度：14%
```

## 分支选项
- 步骤1后：可选择跳过某些审计环节
- 步骤6后：可直接开始执行改进任务
- 完成后可跳转回其他流程

## 完成标准
- 代码审计范围明确
- 静态分析完成
- 代码审查完成
- 安全审计完成
- 性能分析完成
- 审计报告完整
- 改进计划制定

## 失败策略
- 审计过程中可随时暂停
- 支持调整审计范围
- 提供分阶段审计选项

## 适用边界

### 适合场景
- 代码质量审查、安全审查、性能审查、规范检查

### 轻量处理（可直接自动化）
- 已知问题的简单修复

### 需人工介入
- 安全策略、高风险变更
